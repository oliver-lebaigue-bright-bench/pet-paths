# Contributing to Pet Paths

Thank you for your interest in contributing.  
Pet Paths is a lightweight, static frontend project, and all contributions are appreciated.

Before contributing, please review:  
- **TERMS_OF_SERVICE.md**  
- **ACCEPTABLE_USAGE_POLICY.md**  
- **SECURITY.md**  
- **PRIVACY.md**

---

## How to Contribute

### 1. Fork the Repository
Create a fork of the repository under your own GitHub account.

### 2. Make Focused Changes
Keep pull requests small and specific. Examples:
- Fixing a bug  
- Improving UI behaviour  
- Updating documentation  
- Adding small features that fit the projectâ€™s direction  

Avoid large rewrites without discussion.

### 3. Include Visuals for UI Changes
If your PR affects the layout or design, include screenshots.

### 4. Keep the Code Simple
The project uses:
- HTML  
- CSS  
- JavaScript  
No build tools or frameworks.

Please match the existing style.

### 5. Do Not Submit Sensitive Data
Never commit:
- API tokens  
- Firebase service accounts  
- Credentials  
- Personal information  

### 6. Open a Pull Request
Describe:
- What you changed  
- Why the change is needed  
- Any steps needed to test it  

A maintainer will review it as time permits.

---

## Reporting Bugs

Before submitting a bug:
- Check the Issues tab to avoid duplicates  
- Provide clear reproduction steps  
- If security-related, follow **SECURITY.md** (email only)

---

## Feature Requests

You may open an Issue to propose a new feature.  
Include context, reasoning and minimal examples.

Not all features will be accepted; Pet Paths aims to stay lightweight.

---

## Code of Conduct

Participation is governed by **CODE_OF_CONDUCT.md**.

Thank you for helping improve Pet Paths.
